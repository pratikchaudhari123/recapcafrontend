========================================
 HOW IT WORKS
========================================

1. USER VISITS FRONTEND
   └─> https://your-app.vercel.app
   └─> Vercel serves Next.js app

2. USER LOGS IN
   └─> Firebase Authentication
   └─> Token stored in browser

3. USER MAKES API CALL
   └─> Frontend: fetch('/api/subscription/status')
   └─> Vercel Proxy: http://13.201.103.113/api/subscription/status
   └─> EC2 Backend: Processes request
   └─> Response: Returns data
   └─> Frontend: Displays data

4. USER MAKES PAYMENT
   └─> Frontend: Razorpay checkout
   └─> Razorpay: Processes payment
   └─> Frontend: Sends verification to /api/payment/verify
   └─> Vercel Proxy: Routes to EC2
   └─> EC2 Backend: Verifies and activates subscription
   └─> Response: Success
   └─> Frontend: Shows success message

========================================
 DATA FLOW EXAMPLES
========================================

EXAMPLE 1: Get Subscription Status
-----------------------------------
Browser → Vercel
  GET /api/subscription/status
  Headers: Authorization: Bearer <token>

Vercel → EC2
  GET http://13.201.103.113/api/subscription/status
  Headers: Authorization: Bearer <token>

EC2 → MongoDB
  Query user subscription

MongoDB → EC2
  Return subscription data

EC2 → Vercel
  Response: { plan_type: "premium", is_active: true, ... }

Vercel → Browser
  Response: { plan_type: "premium", is_active: true, ... }

Browser
  Display subscription status


EXAMPLE 2: AI Chat
------------------
Browser → Vercel
  POST /api/ask-question
  Body: { question: "What is CA?" }
  Headers: Authorization: Bearer <token>

Vercel → EC2
  POST http://13.201.103.113/api/ask-question
  Body: { question: "What is CA?" }
  Headers: Authorization: Bearer <token>

EC2 → OpenAI
  Query with context from FAISS

OpenAI → EC2
  Return AI response

EC2 → Vercel
  Response: { answer: "CA stands for...", category: "general" }

Vercel → Browser
  Response: { answer: "CA stands for...", category: "general" }

Browser
  Display AI response in chat


EXAMPLE 3: Payment
------------------
Browser → Razorpay
  Create order

Razorpay → Browser
  Return order_id

Browser → Vercel
  POST /api/payment/create-order
  Body: { plan_type: "premium" }
  Headers: Authorization: Bearer <token>

Vercel → EC2
  POST http://13.201.103.113/api/payment/create-order
  Body: { plan_type: "premium" }
  Headers: Authorization: Bearer <token>

EC2 → Razorpay
  Create order

Razorpay → EC2
  Return order details

EC2 → Vercel
  Response: { order_id: "...", amount: 999, ... }

Vercel → Browser
  Response: { order_id: "...", amount: 999, ... }

Browser → Razorpay
  Show payment modal

User → Razorpay
  Complete payment

Razorpay → Browser
  Return payment_id, signature

Browser → Vercel
  POST /api/payment/verify
  Body: { razorpay_order_id, razorpay_payment_id, razorpay_signature, plan_type }
  Headers: Authorization: Bearer <token>

Vercel → EC2
  POST http://13.201.103.113/api/payment/verify
  Body: { razorpay_order_id, razorpay_payment_id, razorpay_signature, plan_type }
  Headers: Authorization: Bearer <token>

EC2 → Razorpay
  Verify signature

EC2 → MongoDB
  Update user subscription

EC2 → Vercel
  Response: { success: true, message: "Payment verified", ... }

Vercel → Browser
  Response: { success: true, message: "Payment verified", ... }

Browser
  Show success message, redirect to dashboard
