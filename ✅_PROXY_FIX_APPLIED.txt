 💡 WHY THIS FIX
========================================

OLD METHOD (vercel.json):
- Used Vercel's rewrite configuration
- Sometimes causes mixed content issues
- Less control over proxy behavior

NEW METHOD (Next.js API Route):
- Native Next.js API route
- Better control over requests/responses
- Handles CORS automatically
- More reliable on Vercel
- No mixed content errors

========================================
 🔍 HOW IT WORKS
========================================

Frontend Request:
fetch('/api/subscription/status')

Next.js API Route:
app/api/proxy/route.ts intercepts

Proxy Logic:
1. Receives request at /api/*
2. Replaces origin with backend URL
3. Forwards to: http://13.203.232.46:8000/api/*
4. Returns response to frontend

Backend Response:
Returns data from EC2 backend

Result:
✓ No CORS errors
✓ No mixed content errors
✓ HTTPS frontend → HTTP backend works

========================================
 📋 PROXY ROUTE CODE
========================================

File: app/api/proxy/route.ts

export async function GET(req: Request) {
  const url = req.url.replace(
    req.nextUrl.origin + "/api", 
    "http://13.203.232.46:8000/api"
  );
  const response = await fetch(url, {
    headers: req.headers,
  });
  return new Response(await response.text(), {
    status: response.status,
    headers: response.headers,
  });
}

export async function POST(req: Request) {
  const body = await req.text();
  const url = req.url.replace(
    req.nextUrl.origin + "/api", 
    "http://13.203.232.46:8000/api"
  );
  const response = await fetch(url, {
    method: "POST",
    headers: req.headers,
    body,
  });
  return new Response(await response.text(), {
    status: response.status,
    headers: response.headers,
  });
}
